# court-format-converter

裁判所提出書面（準備書面・訴状・答弁書等）を裁判所実務の書式慣行に準拠した体裁に一括整形するツールです。

岡口マクロ（[VBAソース](https://www.slaw.jp/2015/08/ms-wordvba.html)）を基盤に、括弧付き番号の1文字統一（`⑴`）、動的インデント計算、番号後スペース正規化などの改良を加えた独自のインデントシステム（v7方式）を採用しています。開発経緯は [HANDOFF.md](HANDOFF.md) に記録しています。

**2つの使い方があります:**
- **Python CLI版** — docxファイルをコマンドラインで一括変換
- **Word アドイン版** — Word上でボタン1つで書式整形（Windows / Mac対応）

文化審議会建議「公用文作成の考え方」（令和4年1月7日）および裁判所実務における書式慣行に基づいています。

## 背景

裁判所に提出する書面には、ページ設定・フォント・見出し番号体系・インデント等について実務上の統一的な書式慣行があります。しかし、Wordで手作業で整形するのは煩雑で、特に他のソフトウェアで作成された文書や、書式の統一されていない文書の整形には時間がかかります。

このツールは、docxファイルを読み込み、裁判所書式に必要な整形を自動で行います。

## 機能

### ページ設定
- A4縦（210mm x 297mm）
- 余白: 上35mm / 下25mm / 左30mm / 右20mm
- 26行 x 37文字（全角基準）のグリッド設定
- フッターにページ番号（中央配置）

### フォント統一
- 日本語: MS 明朝 12pt
- 英数字: Times New Roman 12pt
- テーブル内: 10pt

### 見出し自動検出とインデント

テキストパターンから見出し階層を自動判定し、全角1字刻みのインデントを適用します。
岡口マクロ（VBAソース: [slaw.jp](https://www.slaw.jp/2015/08/ms-wordvba.html)）を参考に、以下の改良を加えています。

| レベル | 番号形式 | 左インデント | 開始位置 |
|---|---|---|---|
| 1 | 第１、第２… | 0字 | 0字目 |
| 2 | １、２… | 1字 | 1字目 |
| 3 | ⑴、⑵… | 2字 | 2字目 |
| 4 | ア、イ… | 3字 | 3字目 |
| 5 | （ア）、（イ）… | 4字 | 4字目 |
| 6 | ａ、ｂ… | 5字 | 5字目 |
| 7 | ⒜、⒝… | 6字 | 6字目 |

本文段落は `見出しleft + 番号文字数 + 全角スペース1個` の位置から開始します。番号の文字数に応じて動的に計算されるため、`⑴`（1文字）でも `（ア）`（3文字）でも正しい位置に揃います。

#### 岡口マクロとの違い

| | 岡口マクロ | 本ツール |
|---|---|---|
| 括弧付き番号 | `(1)` 3文字（インデントがずれる） | `⑴` 1文字に統一（ずれない） |
| 本文インデント | 固定値 | 番号文字数から動的計算 |
| 番号後スペース | 手動 | 全角1個に自動正規化 |
| hanging indent | 使用（Wordの文字幅計算でずれる場合あり） | 不使用（leftのみ）|
| 丸数字変換 | なし | `①⑴（１）` → `⑴` に統一 |

#### インデントの既知の制限

Wordの段落モデルでは、1つの段落内で「1行目」と「2行目以降」の左位置を完全に独立して制御できません。そのため、以下のケースでは手動修正が必要です:

- **コミ出しなし（本文兼用）の見出し行が長くて折り返す場合**: 2行目が見出し番号の直下に来ることがあります。裁判所の判決文のように番号の下を完全に空けるには、段落を手動で分割する必要があります。

これは岡口マクロでも同様の制限があり、Wordのインデントシステムの本質的な限界です。

### 見出し番号の自動再付番

文書の番号体系が裁判所書式と異なる場合、自動的に正しい番号体系に変換します。

```
変換前（公文書スタイル）:       変換後（裁判所書式）:
１　走行履歴の出典        →    第１　走行履歴の出典
(1)　データの形式         →    １　データの形式
(2)　データの取得方法     →    ２　データの取得方法
ア　コピー手順            →    (1)　コピー手順
２　走行記録の分析        →    第２　走行記録の分析
```

上位レベルが進むと下位のカウンターは自動でリセットされます。

### 半角→全角変換
- 数字・英字・カタカナ・括弧・記号を全角に統一
- 半角カタカナの濁点・半濁点を正しく合成
- テーブル内も変換対象

### その他
- 冒頭セクション（事件番号・当事者表示・書面タイトル・日付・署名）は自動判定してインデントを維持
- 「以上」等の定型句は右寄せ

## Python CLI版

### インストール

```bash
pip install python-docx
```

### 使い方

```bash
# 基本（出力ファイル名は自動生成）
python3 court_format_converter.py 準備書面.docx

# 出力ファイル名を指定
python3 court_format_converter.py 準備書面.docx 準備書面_整形済.docx
```

出力ファイル名を省略した場合、`<元ファイル名>_裁判所書式.docx` として保存されます。

## Word アドイン版

Wordのリボンに「裁判所書式」ボタンを追加し、開いている文書をその場で整形します。Python不要。

### セットアップ（Mac）

1. [manifest.xml](word-addin/manifest.xml) をダウンロード
2. 以下のフォルダにコピー（`wef` フォルダがなければ作成）:
   ```
   ~/Library/Containers/com.microsoft.Word/Data/Documents/wef/
   ```
3. Wordを起動（既に開いている場合は再起動）
4. **挿入** → **アドイン** → **マイ アドイン** → 「裁判所書式変換」を選択

### セットアップ（Windows）

1. [manifest.xml](word-addin/manifest.xml) をダウンロード
2. Windowsの共有フォルダに配置（例: `\\localhost\c$\manifests\`）
3. Wordを開く → **ファイル** → **オプション** → **セキュリティ センター** → **信頼されたアドイン カタログ** に共有フォルダのURLを追加
4. **挿入** → **アドイン** → **マイ アドイン** から使用

### 使い方

1. Wordで書面を開く
2. リボンの **裁判所書式** ボタンをクリック
3. 適用する項目を選択して **変換を実行**

変換項目は個別にON/OFFできます:
- ページ設定（A4・余白・行数・文字数）
- フォント統一（MS 明朝 12pt）
- 見出し検出・インデント適用
- 半角→全角変換
- ページ番号（フッター中央）

## 対応する書面の種類

以下の書面で動作確認しています:

- 準備書面
- 訴状・答弁書
- 報告書
- 意見書
- 上申書
- 陳述書

見出し番号体系（第１→１→(1)→ア→…）に従った文書であれば、書面の種類を問わず整形できます。

## 注意事項

- 文章の内容は一切変更しません（書式整形のみ）
- 元のdocxファイルは変更されません（別ファイルに出力）
- 画像・図表の位置は保持されますが、レイアウトが崩れる場合があります
- 複雑なスタイルが設定された文書では、一部の書式が正しく変換されない場合があります

## 書式の根拠

本ツールの書式設定は以下に基づいています:

- **文化審議会建議「公用文作成の考え方」**（令和4年1月7日）
- **裁判所実務における書式慣行**（余白・行数・文字数・フォント・番号体系）

## 今後の予定

現在の書式整形機能に加えて、以下の機能を開発中です。

### AI構造化モード（開発中）

見出し番号のない文書や、番号体系が崩れた文書に対して、LLM（大規模言語モデル）が文書構造を自動判定し、適切な見出し番号を付与します。

### 公用文変換（開発中）

文化審議会建議に基づく公用文の作法に沿って、文体を自動整形します。冗長表現の排除、一文一論点への分割、修飾語の適切な配置などを行います。

### 陳述書ドラフト生成（開発中）

打ち合わせメモや議事録から、裁判所提出用の陳述書ドラフトを自動生成します。事件類型（離婚・刑事・労働・交通事故等）に応じたテンプレートで構成します。

> これらのAI機能はLLMのAPI利用料が発生するため、有料プランでの提供を予定しています。書式整形機能（現在の機能）は引き続き無料でご利用いただけます。
