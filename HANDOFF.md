# court-format-converter 開発経緯

## プロジェクト概要

裁判所提出書面（準備書面・訴状等）のdocxを裁判所書式に整形するツール。
Python CLI版とWord アドイン版（Office.js）の2版がある。

- リポジトリ: https://github.com/yutoribengoshi/court-format-converter
- Python版: `court_format_converter.py`
- アドイン版: `word-addin/src/converter.js`

## インデント設計の経緯（2026-04-06）

### 最終到達点: v7方式

岡口マクロ（VBAソース: https://www.slaw.jp/2015/08/ms-wordvba.html ）を基盤としつつ、以下の改良を加えた独自方式。

#### 見出しインデント

| レベル | 番号 | left | 開始位置 |
|---|---|---|---|
| 1 | 第１ | 0字 | 0字目 |
| 2 | １ | 1字 | 1字目 |
| 3 | ⑴ | 2字 | 2字目 |
| 4 | ア | 3字 | 3字目 |
| 5 | （ア） | 4字 | 4字目 |
| 6 | ａ | 5字 | 5字目 |
| 7 | ⒜ | 6字 | 6字目 |

- hanging indentは使わない（Wordの文字幅計算でずれるため）
- コミ出しなし（本文兼用）の見出しが長い場合は `firstLine=1` を適用

#### 本文インデント

`body_start = heading_left + 番号文字数 + 1（全角スペース）`

- 1行目: `body_start` の位置から
- 2行目以降: `body_start - 1` の位置から（`left = body_start - 1, firstLine = 1`）

番号文字数は動的計算:
- `⑴` = 1文字 → body_start = 2 + 1 + 1 = 4
- `（ア）` = 3文字 → body_start = 4 + 3 + 1 = 8
- `第１` = 2文字 → body_start = 0 + 2 + 1 = 3

### 岡口マクロとの比較

| | 岡口マクロ | v7方式 |
|---|---|---|
| 括弧付き番号 | `(1)` 3文字 → インデントずれ | `⑴` 1文字に統一 → ずれない |
| 本文インデント | 固定値 | 番号文字数から動的計算 |
| 番号後スペース | 手動 | 全角1個に自動正規化 |
| hanging indent | 使用（ずれる場合あり） | 不使用（leftのみ） |
| 丸数字変換 | なし | `①⑴（１）(1)` → `⑴` に統一 |

### 既知の制限

**コミ出しなしの見出しが長くて折り返す場合、2行目が見出し番号の直下に来る。**

裁判所の判決文のように番号の下を完全に空けるには、段落を手動で分割する必要がある。これはWordのインデントモデルの本質的な制限であり、岡口マクロでも同様。

### 試行錯誤の記録

v7に至るまで以下を試行:

1. **元のdocxを直接編集** → 元のグリッド設定でtwips値がずれる
2. **新規docx作成＋テキスト流し込み** → グリッド依存を排除。採用
3. **twips直値のみ（Charsなし）** → Wordが保存時にCharsに変換してずれる
4. **Chars属性のみ（twipsなし）** → 一部環境で無視される
5. **Chars + twips両方設定** → 採用。岡口マクロと同じ方式
6. **hanging indent** → Wordの文字幅計算でずれる。廃止
7. **leftのみ + firstLine** → 採用。ずれない
8. **`（１）` 3文字問題** → `⑴` 1文字に統一して解決
9. **`（ア）` 3文字問題** → 動的body_indentで対応（1文字版なし）
10. **firstLine=0 vs firstLine=1** → 本文は `firstLine=1`（2行目が1字左）が正解
11. **見出し行の折り返し** → コミ出しなしの長い見出しにも `firstLine=1` 適用

### ルール要約

1. 全角1字刻みで階層が深くなる
2. 見出し番号の直後に全角スペース1個（自動正規化）
3. 本文の1行目は見出し番号＋スペースの直後から
4. 本文の2行目以降は1行目の1字左（吊り下げ的だがfirstLineで実現）
5. 括弧付き番号は1文字版（`⑴⒜`）に統一
6. `⑴` → 20個まで。21以降は `（２１）` にフォールバック
7. `（ア）` は1文字版がないため3文字のまま。動的計算で対応

## LLM機能（有料版候補）

- `--auto`: LLM構造化（見出し番号自動付与）
- `--auto --tone`: 公用文変換の修正候補をWordコメントで提示（原文不変）
- `--fix-numbering`: 連番ミス検出・修正
- `--check`: 事実摘示チェッカー（主語欠落・構文順序をWordコメントで指摘）
- `--provider`: Anthropic / Ollama / OpenAI互換

### 原則

**原文は黙って書き換えない。** 指摘はWordコメント、書き換えは変更履歴（Track Changes）。
